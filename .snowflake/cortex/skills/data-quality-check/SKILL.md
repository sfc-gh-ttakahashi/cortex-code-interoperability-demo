# データ品質チェック

指定されたテーブルに対して以下の品質チェックを実行してください。

## チェック項目
1. 各カラムの NULL 率を計算する
2. 主キー候補カラムのユニーク率を確認する
3. 数値カラムの MIN / MAX / AVG を算出する
4. DATE / TIMESTAMP カラムの範囲と欠損日を特定する
5. STRING カラムの空文字率を確認する

## 出力フォーマット

チェック結果は以下の形式でマークダウンテーブルとして報告してください。

### 判定基準
- **OK**: 問題なし（NULL率 < 1%, ユニーク率 100%）
- **WARN**: 要確認（NULL率 1-5%, ユニーク率 95-99%）
- **NG**: 問題あり（NULL率 > 5%, ユニーク率 < 95%）

### 出力例

| カラム名 | 型 | NULL率 | ユニーク率 | 判定 | 備考 |
|---------|------|--------|-----------|------|------|
| order_id | INT | 0% | 100% | OK | 主キー候補 |
| status | STRING | 0% | 30% | OK | カテゴリ値 |

## 注意事項
- テーブルの COMMENT が設定されている場合は冒頭に表示する
- 行数が多い場合は TABLESAMPLE を使って効率化する
